## üìã Documentation Standards

When creating new documentation:

1. **Location**:
   - How-to guides ‚Üí `guides/`
   - Process/methodology ‚Üí `methodology/`
   - Analysis/reference ‚Üí `reference/`

2. **Format**:
   - Use clear markdown headers
   - Include table of contents for long docs
   - Add cross-references to related docs
   - Use code blocks for commands/examples

3. **Naming**:
   - UPPERCASE for doc files (e.g., `SITE-EDITING-GUIDE.md`)
   - Descriptive names (avoid abbreviations)
   - Include category in name when helpful

4. **Updates**:
   - Keep this index updated when adding new docs
   - Update main README.md if major documentation added
   - Add "Last Updated" date to long-living guides
